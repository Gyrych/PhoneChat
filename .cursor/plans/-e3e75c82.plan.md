# 在主页面侧边栏中让会话记忆窗格独占抽屉宽度

我将按照你选择的 a)：在主页面抽屉（`.drawer`）内，使会话记忆窗格展开时独占整行，宽度等于抽屉可用宽度（跟随 `--drawer-width`），管理按钮保持不变（在摘要上方或抽屉其它位置）。

要修改的文件：

- `index.html`（或 `dist/index.html`，视项目运行分支） — 检查侧边栏内会话/分组记忆 DOM 位置（不会改 DOM，只读验证）。
- `style.css` — 在抽屉作用域下增加/调整样式，确保仅影响侧边栏内的记忆窗格。
- `dist/style.css` — 同步产物。
- `CURSOR.md` — 追加变更记录与目的说明（同步文档策略）。

关键实现（将在 `style.css` 中添加/调整）：

```css
/* 仅影响抽屉内的会话/分组记忆显示，使其独占一行并跟随抽屉宽度 */
.drawer .conversation-summary-content,
.drawer .group-memory {
  flex-basis: 100%; /* 换行并占满整行 */
  width: 100%;
  box-sizing: border-box;
  margin: 6px 0;
}
/* 允许抽屉内的会话项换行，以便摘要独占一行，不影响页面其他地方 */
.drawer .drawer-item {
  display: flex;
  flex-wrap: wrap;
  align-items: flex-start;
}
.drawer .drawer-item .drawer-item-main {
  flex: 1 1 auto; /* 保持主要信息可收缩 */
  min-width: 0;
}
```

注意：我会限定选择器到 `.drawer` 作用域，避免影响 `conversations.html`（会话管理页）中相同类名的行为。执行前会先读取 `index.html`（和 `dist/index.html`）确认侧边栏 DOM 结构以保证选择器准确。

验收标准：

- 在主页面打开抽屉并展开会话/分组，点击“查看会话记忆”时，记忆窗格应换行并占满抽屉内容宽度（与分组记忆一致），左右内边距与分组记忆一致。
- 其他页面（例如 `conversations.html`）的布局不变。

回滚说明：删除新增的 `.drawer` 作用域 CSS 规则并恢复 `CURSOR.md` 即可回退。
